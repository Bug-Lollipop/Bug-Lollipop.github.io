Font 和Typeface 的区别

Font 和Typeface
今天对 Font 和 Typeface 这两个比较容易混淆的概念了解了一下，看了知乎的 英语术语「typeface」和「font」的区别是什么？
然后又看了一片外文： Font vs Typeface? What’s the Difference?

大体区别如下：
typeface是指整个字体设计的集合/系列（style，weight，variation）。 像Georgia(衬线字体)，Helvetica（无衬线字体），FF Meta和Chaparral Pro(英文衬线字体)。 这些是不同的字体，可以称为字体家族。

font指的是原始字体设计的版本，例如 字体样式，尺寸，粗细等。

知乎上的一句总结很到位：

方正兰亭黑是 typeface
方正兰亭纤黑是 style
五号、regular的方正兰亭纤黑是 font

input 输入框的步长问题

问题：在使用 <input type="number">输入框时，如果输入了当前输入框中值不以1为单位增减的值时会出现：请输入有效值 两个最接近的有效分别为...的提示，这是因为浏览器默认认为<input type="number">的步长为1。

解决办法：指定<input type="number">输入框的步长，如果增加属性step="0.01"，可以根据具体的精度调整step属性的值。

优雅的增加点击区域的大小

移动端我们点击屏幕上很小的箭头时候，经常会很费经，要不就点不到，要不就误点到其他的地方，正常我们都是按照这个图标的原始尺寸写的，例如：

.icon-arrow {
  width: 16px;
  height: 16px;
}

效果没问题，但是肯定体验不友好，因为尺寸仅仅16px，我们很容易点不中。
稳妥的方法是外部在嵌套一层标签，专门控制点击区域大小，如果对代码要求比较高，则可以使用padding 或透明的border增加元素的点击区域大小。

其中首推透明 border方法，原因是如果我们的图标是工具生成的，那么background-position 就是限定死的值，若用 padding 撑开间距，就会遇到定位不准的问题。但是若用透明的border 增加点击区域，就没这个问题，只要合并时留下足够的间距就可以了，例如：

.icon-arrow {
  width: 16px;
  height: 16px;
  border: 11px solid transparent;
}

没必要的 currentColor 变量

我们知道 currentColor 变量是个好东西，可以使用当前 color 计算值，IE9 + 都能支持。
实际上， CSS 中很多属性值默认就是 currentColor的表现，我们一般 无需再画蛇添足去再声明这个关键字。如 border 、 text-shadow、 box-shadow 等，尤其是 border。
因此有需要元素 hover 改变颜色和边框的情况，只需要控制 color 的值就可以了，例如：

.btn {
  color: red;
  border: 2px solid;
}

.btn:hover {
  color: blue;
}

而不需要去这样加：
border: 2px solid currentColor

CSS writing-mode 的更多使用

writing-mode 通常可以用来实现文字的竖向呈现。
属性对应的关键字有下面几个：

writing-mode: horizonal-tb;  /* 默认值，表示文本流是水平方向的，元素是从上向下的*/
writing-mode: vertical-rl;  /* 表示文本是垂直方向展示 ，阅读顺序是从右向左（right-left）*/
writing-mode: vertival-lr;  /* 表示文本是垂直方向展示 ，阅读顺序是从左向右（left-right）*/

除了竖直排版的时候需要writing-mode，其实还可以实现很多其他的效果。

writing-mode 将页面默认的水平文档流改成了垂直流，基于水平方向的规则都可以适用于垂直方向，比如：

  • margin 合并
  • 用 text-align: center 实现图片垂直居中
  • 使用 text-indent 实现文字下沉效果
  • 可以实现全兼容的 icon-fonts 图标的旋转效果

具体的各个效果可以看这个 http://demo.cssworld.cn/ 中的 （12-2-1） ~ （12-2-5） 里面的代码效果
