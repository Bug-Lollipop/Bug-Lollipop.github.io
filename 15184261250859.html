HTML5 桌面通知：Notification API

最近授权 gitlab Notification 通知栏，感觉还不错。

相比传统的页面内嵌入固定定位的div来展示通知的方式，Notification的一大特色在于：即便用户在浏览其他页卡或将浏览器最小化，仍然可以及时收到桌面通知。因此工作时也可以收到最新的新闻、新邮件的提醒等等。

也正是由Notification有如此高的优先级，所以浏览器出于安全限制，对通知的弹出增加了用户的授权。

用户权限

Notification.permission 该属性用于表明当前通知显示的授权状态，可能的值包括：
- default ：不知道用户的选择，默认。
- granted ：用户允许。
- denied ：用户拒绝。

if (Notification.permission === 'granted') {
    console.log('用户允许通知');
} else if (Notification.permission === 'denied') {
    console.log('用户拒绝通知');
} else {
    console.log('用户还没选择，去向用户申请权限吧');
}

请求权限

Notification.requestPermission (function(permission) {
if (permission === 'granted') {
      // 允许
} else if (permission === 'denied') {
     // 禁止
}
});

推送通知

获取用户授权之后，就可以推送通知了。

var notification = new Notification(title, options)

关闭通知

var notification = new Notification(title, options)

setTimeout (function () {
    notification.close();
}, 3000);

应用场景

当被很多无太大意义的邮件提醒搞烦而去关闭邮件提醒的时候，说明这个网站可能会适合加Notification。不再需要频繁的点击邮件，查看邮件，删除邮件等操作。
- 各种消息推送
- 新闻网站
