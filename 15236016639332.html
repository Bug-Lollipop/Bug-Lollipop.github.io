【译】如何逃离 async / await 地狱

async/ await 使我们从回调地狱中解脱出来。但是人们又开始被它虐待 - 进入了 async/ await 地狱

在这片文章中，我将会尝试去解释 async/await 地狱是什么，同时分享一些解决方案。

什么是 async/await 地狱

当 Javascript 异步编程的时候时，人们总是使用很多 await 语句，并在函数调用之前总是等待它。这会导致性能问题，因为一个语句多次调用并不依赖前一个语句 - 但是你仍然必须等待前一个语句的完成。

一个 async/ await 地狱的例子

尝试去写一个去实现购买披萨和饮料的功能。这个代码可能是这个样子的：

(async () => {
  const pizzaData = await getPizzaData()    // async call
  const drinkData = await getDrinkData()    // async call
  const chosenPizza = choosePizza()    // sync call
  const chosenDrink = chooseDrink()    // sync call
  await addPizzaToCart(chosenPizza)    // async call
  await addDrinkToCart(chosenDrink)    // async call
  orderItems()    // async call
})()
