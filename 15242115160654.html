前端实现图片上传

移动端

移动端的H5通过 JsBridge 上传图片，返回base64，然后我们需要将base64 转换成Blob 二进制文件，再发送给后端，代码如下：


export const convertBase64UrlToBlob = dataURI => {
  const arr = dataURI.split(',')
  const mime = arr[0].match(/:(.*?);/)[1]
  
  //去掉base64中的换行符，webkit会自动去除，但是IOS9以及IOS8中不会自动去除，导致转换出错
  const bstr = atob(arr[1].replace(/\s/g, ''))
  let n = bstr.length
  const u8arr = new Uint8Array(n)
  while (n--) {
    u8arr[n] = bstr.charCodeAt(n)
  }
  return new Blob([u8arr], { type: mime })
}

PC端

PC 不同之处在于要自己去获取图片数据：
首先通过input：file 标签获取File 对象，使用FileReader对象的readAsDataURL方法 读取file 获得Base64 编码，将Base64编码转换成Blob 二进制文件，最后构造 FormData 数据。

创建 File 标签

<input class="upload-file" name="File" type="file" @change="handleFileChange">


handleFileChange (e) {
  let files = e.target.files[0]
  // 预览
  this.imgPreview(files)
  if (files.length === 0) return
  files.forEach((item, index) => {
    let formData = new FormData()
    formData.append('file', item)
    API.uploadFiles(formData).then(res => {
      // ...上传成功后的操作
    }).catch(err => {
     // error
    }
 })
},

//上传的图片转成base64位码
imgPreview (files) {
    if (!files || !window.FileReader) return

        reader.onload = (e) => {

          // 限制图片大小
          if (mb > 10) {

          var obj = {

        }
        reader.readAsDataURL(files)
