SSH 多账户使用

参考github 的 Connecting to GitHub with SSH

检查 SSH Keys 是否存在

ls -al ~/.ssh

如果不存在，创建一个新的 SSH key

ssh-keygen -t rsa -b 4096 -C "your_email@example.com"

在 “-t” 参数之后，请求建立一个 “RSA” 类型的密钥。
RSA 是当前最新并且最安全的一种形式。在 “-C” 参数之后，提供了一个注释，可以把它想象为对这个密钥的一种描述或标签。例如使用email 地址。总之，一个能更容易识别的注释。

在确认此命令后，会询问一些信息：

(1) 给这个新的密钥输入一个名称，保留默认的名字和设置。
(2) 提供一个密码。虽然 SSH 公钥可以确保在没有任何密码的情况下安全地使用，但是还是应该设置一个密码，用来进一步提高安全性。

查看公钥

执行上面命令后，会在.ssh 的目录中找到 id_rsa.pub（公钥）， id_rsa（秘钥）文件，如果想看公钥里面的内容，可以使用命令：

cat ~/.ssh/id_rsa.pub

将SSH Key 添加到 ssh-agent 中

  • 后台启动ssh-agent
eval“$（ssh-agent -s）”
  • 如果使用的是macOS Sierra 10.12.2或更高版本，则需要修改~/.ssh/config文件以自动将密钥加载到ssh-agent中，并将密钥存储在钥匙串中。
  • 修改config 文件, 如果没有先创建一个 touch ~/.ssh/config 配置如下：
Host [shortName] // 简称
HostName [domain] // 对应服务器的域名
PreferredAuthentications publickey
IdentityFile [private keyPath] // 私钥绝对路径
User [user] // 用户名。服务器的用户名，不是你的用户名。eg：大部分的git服务器用户名都为git
Port [port] // 端口号，默认为22

比如如果在 github 上有一个账号，在gitlab上也有个账号，那么config 下的文件内容可以为：

Host github
HostName github.com
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa_github
User git
    
Host gitlab
HostName gitlab.elenet.me
PreferredAuthentications publickey
IdentityFile ~/.ssh/id_rsa)gitlab
User git
  • 验证是否成功 单个SSH 验证，可以直接 ssh -T git@github.com 进行验证，但是设置了config 之后，是通过简称映射对应的域名和私钥， 命令为：
ssh -T git@github
ssh -T git@gitlab
  • 将SSH私钥添加到ssh-agent并将密码存储在钥匙串中。 如果使用不同的名称创建密钥，或者要添加具有不同名称的现有密钥，请将该命令中的id_rsa替换为私钥文件的名称。
ssh-add -K〜/ .ssh / id_rsa

注意： 该-K选项是Apple的标准版本ssh-add，当将ssh密钥添加到ssh-agent时，该密钥将密钥存储在您的钥匙串中。

复制公钥内容

pbcopy < ~/.ssh/id_rsa.pub

补充：
ssh-keygen 的常用参数：
* -b ：指的是位数，默认2048
* -C ：提供一个新注释，自定义一个邮箱
* -c ：要求修改私钥和公钥文件中的注释。本选项只支持 RSA1 密钥。程序将提示输入私钥文件名、密语(如果存在)、新注释。
* -f ：指定密钥文件名。
* -N 创建秘钥的密码
* -q ：安静模式。用于在 /etc/rc 中创建新密钥的时候。
* -t ：指定要创建的密钥类型。
